# 登录功能使用说明

## 功能特性

✅ **真正的 Netlify Identity 登录**
- 使用 Netlify 官方的身份验证服务
- 支持邮箱注册和登录
- 安全的 JWT 令牌认证

✅ **云端数据同步**
- 登录时自动保存所有配置到云端
- 跨设备同步您的数据
- 退出登录时自动清除本地数据

✅ **自动保存**
- 数据变化时自动保存到云端（1 秒延迟）
- 无需手动保存
- 防止数据丢失

✅ **用户头像显示**
- 登录后显示用户头像
- 支持自定义头像或自动生成

## 部署步骤

### 1. 推送到 GitHub

```bash
cd "/Users/pinkocean/Desktop/开发/Round table/ai-forum"
git add .
git commit -m "添加完整的登录功能"
git push origin main
```

### 2. 在 Netlify 启用 Identity

1. 访问 https://app.netlify.com
2. 找到您的 "圆桌" 站点
3. 点击左侧菜单的 **Identity**
4. 点击 **Enable Identity**
5. 在 **Settings and usage** 中配置注册选项：
   - **Open**: 任何人都可以注册（推荐）
   - **Invite only**: 仅邀请用户可以注册

### 3. 测试登录

1. 等待 Netlify 完成部署（约 1-2 分钟）
2. 打开您的 Netlify 站点 URL
3. 点击右上角的 **登录** 按钮
4. 点击 **Sign up** 注册账户
5. 输入邮箱和密码
6. 检查邮箱并验证
7. 登录成功后即可看到头像

## 数据同步流程

### 登录时：
```
用户登录 → 获取 JWT 令牌 → 从云端加载配置 → 恢复到本地存储 → 刷新页面
```

### 使用时：
```
数据变化 → 1 秒延迟 → 自动保存到云端
```

### 退出时：
```
确认退出 → 清除本地数据 → 跳转到登录页
```

## 保存的数据

以下数据会自动同步到云端：

- ✅ 话题列表 (topics)
- ✅ 当前话题 (currentTopicId)
- ✅ 激活的角色 (activeCharacters)
- ✅ AI 配置 (apiConfigs)
- ✅ 对话设置 (dialogSettings)

## 常见问题

### Q: 登录后看不到头像？
A: 确保已在 Netlify 后台启用 Identity 服务

### Q: 数据无法保存？
A: 检查浏览器控制台是否有错误，确保已正确登录

### Q: 可以在多个设备使用吗？
A: 可以！在同一账户登录会自动同步所有数据

### Q: 退出登录后数据会丢失吗？
A: 本地数据会清除，但云端数据保留。再次登录会自动恢复

## 技术实现

- **前端**: Netlify Identity Widget
- **后端**: Netlify Functions (Serverless)
- **存储**: Netlify Blobs
- **认证**: JWT Tokens
