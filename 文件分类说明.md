## 📁 构建产物（不需要上传到 GitHub）

### dist 文件夹 - 由 Vite 自动生成
```
dist/
└── assets/
    ├── index-*.js       # 编译后的 JavaScript
    └── index-*.css      # 编译后的 CSS
```

**说明：**
- `dist/` 是生产构建输出目录
- 运行 `npm run build` 后自动生成
- **不需要**上传到 GitHub
- 已在 `.gitignore` 中忽略

---

## 📁 node_modules（不需要上传）

```
node_modules/            # npm 依赖包
```

**说明：**
- 运行 `npm install` 后自动生成
- **绝对不要**上传到 GitHub
- 已在 `.gitignore` 中忽略

## 📋 上传到 GitHub 前的检查清单

### 必须上传的文件：
- ✅ `src/` 文件夹（所有源代码）
- ✅ `netlify/functions/` 文件夹（所有函数）
- ✅ 所有配置文件（`.gitignore`, `package.json`, `vite.config.js` 等）
- ✅ `README.md`（项目说明）
- ✅ `deploy.sh`（部署脚本）

### 不需要上传的文件/文件夹：
- ❌ `docs/`（开发文档，可选删除）
- ❌ `dist/`（构建产物，已在 .gitignore 中）
- ❌ `node_modules/`（依赖包，已在 .gitignore 中）
- ❌ `.env`（环境变量，如果有的话）

---

## 🚀 推荐操作

1. **删除 docs 文件夹**（或移动到项目外备份）
2. **检查 .gitignore** 确保包含以下内容：
   ```
   node_modules/
   dist/
   .env
   .env.local
   ```
3. **上传到 GitHub**

## 📝 总结

**可以安全删除的文件夹：**
- `docs/` - 所有开发文档和配置指南

**必须保留的核心文件：**
- `src/` - 所有源代码
- `netlify/functions/` - 所有后端函数
- 根目录的所有配置文件

**自动生成的文件夹（已在 .gitignore 中）：**
- `node_modules/` - 依赖包
- `dist/` - 构建产物
